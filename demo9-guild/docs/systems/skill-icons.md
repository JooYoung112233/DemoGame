# 시스템: 스킬/액션 아이콘 가이드

> 다키스트 스타일 액션 시스템(`src/data/actions.js`)의 아이콘 제작용 레퍼런스.
> **6 클래스 × 4 행동(공격 3 + 스킬 1) = 24 플레이어 아이콘** + **적 7유형 × 2~3 행동 = 16 적 아이콘**.
>
> 현 상태: `actions.js` 의 `icon` 필드는 임시 이모지. 중복이 다수(✨ 6곳, 🎯 3곳, 🛡 3곳, 🔥 3곳, 💥 3곳 등)라 전투 로그/툴팁 가독성을 떨어뜨림 → 전용 아이콘으로 교체 필요.
>
> 관련 문서: [class-skills.md](class-skills.md) (구 1스킬 설계, 참고용), [manual-battle-darkest.md](manual-battle-darkest.md) (전투 룰)

## 1. 공통 제작 룰

### 1.1 사이즈 / 포맷
- 캔버스: **64×64 PNG** (UI 표시 32px 기준 2배), Retina 대비 128px도 가능
- 투명 배경, 외곽 1~2px 패딩
- 파일 위치(예정): `demo9-guild/src/assets/icons/actions/<action_id>.png`
  - 예: `warrior_atk1.png`, `mage_skill.png`

### 1.2 타입 색 코딩 (외곽선/배경 글로우)
| 타입 | 외곽선/글로우 | 의도 |
|---|---|---|
| `attack` | 붉/주황 톤 | 적 대상 공격 |
| `skill` | 금/보라 글로우 + 두꺼운 외곽선 | 쿨다운 자원, 시각 위계 1티어 |
| `support` | 녹/청 톤 | 아군 대상 |

→ 슬롯 UI에서 색만 보고도 “자원 스킬 vs 매턴 공격 vs 버프/힐”을 즉시 구분 가능.

### 1.3 쿨다운 게이지 안전 영역
- UI는 아이콘 위에 **원형 회전 게이지**를 덮음
- 식별에 중요한 디테일은 **중앙 60%** 안쪽에 배치
- 모서리 4귀퉁이는 게이지/숫자 라벨에 가려질 수 있음

### 1.4 클래스 톤 (참고 — `units.js` 색상)
| 클래스 | 메인 색 | 보조 톤 제안 |
|---|---|---|
| 전사 (warrior) | `#4488ff` (청) | 강철 회색 + 금 |
| 도적 (rogue) | `#cc44cc` (자) | 검정 + 진홍(피) |
| 마법사 (mage) | `#8844ff` (보) | 청 + 흰 광채 |
| 궁수 (archer) | `#44cc44` (녹) | 갈색 나무 |
| 사제 (priest) | `#ffcc44` (금) | 흰색 + 옅은 금광 |
| 연금술사 (alchemist) | `#44cccc` (청록) | 갈색 가죽 + 형광 녹/적 액체 |

## 4. 작업 우선순위 (제안)

1. **스킬 6개 먼저** — 자원 행동이라 시각 위계가 가장 중요. 임시 ✨ 5개 중복 즉시 해소.
2. **공격 3종 × 6 클래스 = 18개** — 클래스별로 한 번에 묶어 작업 (톤 통일 쉬움).
3. **적 액션 16개** — 회색 톤 통일 + 보스(pitlord) 3종은 별도 디테일.

## 5. 코드 연동

아이콘 PNG가 준비되면 `actions.js` 의 `icon` 필드를 이모지에서 아이콘 키로 전환하거나, 별도 `iconAsset` 필드 추가 → 렌더링 측에서 `this.add.image()` 로 교체.
- 영향 받는 코드: `ManualBattleScene.js` (3 군데, 액션 라벨/툴팁/로그), `DeployScene.js` (행동 가능 체크 라벨).
- 폴백: 에셋 미로드시 기존 이모지 그대로 사용.
