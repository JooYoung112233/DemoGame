# 시스템: 시간 시스템 (하이브리드)

> 두 트랙: **실시간 트랙** (오프라인에서도 흐름) + **게임 내 시간 트랙** (접속 중에만).

## 1. 두 트랙 분리

| 활동 | 트랙 | 이유 |
|---|---|---|
| 활성 파견 | 실시간 | 오프라인 보상 수령 (게임 회전의 핵심) |
| 제작 진행 | 실시간 | 장시간 제작은 오프라인 활용 |
| 피로도 자연 회복 | 실시간 | 휴식은 시간 누적 |
| 마을 이벤트 등장 | 게임 내 | 플레이어 참여 필요 |
| NPC 매물 로테이션 | 게임 내 | 활동성 필요 |
| 소문 갱신 | 게임 내 | 마을 활동의 일부 |
| 호감도 NPC 대화 | 게임 내 | 능동 상호작용 |

## 2. 실시간 트랙

### 2.1 동작 방식

- 모든 활동은 시작 시각 + 완료 시각(epoch ms)을 저장
- 게임 진입 시 `Date.now()` 비교해서 완료된 항목 처리
- 오프라인 중에도 자연히 진행됨

```javascript
// 활성 파견 예시
{
  id: 'exp_001',
  party: ['merc_1', 'merc_3'],
  zone: 'bloodpit',
  level: 3,
  startedAt: 1731750000000,    // Date.now()
  completesAt: 1731750180000,  // +180초 (3분)
  state: 'running'
}

// 진입 시 체크
if (Date.now() >= exp.completesAt) {
  resolveExpedition(exp);  // 결과 처리
}
```

### 2.2 오프라인 진행 한도

- 누적 한도: **최대 4시간**
- 4시간 넘게 비접속 시 4시간까지만 진행
- 이유: 영구 잠수 방지, 활동 동기 유지
- 길드회관 D(자동화) 후반 업그레이드로 한도 +1시간씩 확장 (최대 8시간)

### 2.3 피로도 회복

```
기본: +1/분 (실시간)
길드회관 C 풀: +5/분 (실시간)
```

오프라인 4시간 = 최대 +240 회복 (실제론 100 캡)

## 3. 게임 내 시간 트랙

### 3.1 마을 이벤트 트리거

- 마을 진입 후 누적 접속 시간 기준
- 30분마다 마을 이벤트 1개 발동
- 길드회관 E(정보) 업그레이드로 빈도 단축 가능 (E3: -30%)

### 3.2 NPC 매물 로테이션

- 마을 누적 접속 시간 기준 매 시간 1회 갱신
- 호감도 높은 NPC는 더 자주 갱신
- 길드회관 E3 적용 시 30분마다

### 3.3 게임 내 시계 속도

- **1:1** (게임 내 시간 = 실시간 접속 시간)
- 별도 가속 없음 (단순함 우선)

## 4. 12시간 페이싱 가이드 연동

| 페이싱 단계 | 누적 접속 시간 | 주요 이벤트 |
|---|---|---|
| 튜토리얼 | 0:00 ~ 1:00 | BP Lv1-2, 첫 NPC 호감 |
| 초반 확장 | 1:00 ~ 2:30 | 제작 해금, BP Lv3-5 |
| 중반 진입 | 2:30 ~ 4:00 | 경매장, Cargo 첫 도전 |
| 중반 가속 | 4:00 ~ 6:00 | BP Lv8, Cargo Lv3 |
| 후반 진입 | 6:00 ~ 8:00 | BO 첫 도전, BP 보스 |
| 후반 확장 | 8:00 ~ 10:00 | Cargo 보스, 모든 구역 동시 가동 |
| 후반 가속 | 10:00 ~ 11:30 | BO 보스, 제작 전설 |
| 엔드게임 | 11:30 ~ 12:00 | 최종 빌드 완성 |

오프라인 시간은 누적에 포함 안 됨 (게임 내 시간 기준).

## 5. UI 표시

### 5.1 HUD 시간 표시

```
상단 우측:
  접속 시간: 02:34:15
  활성 파견: 2/3 (가장 빠른 완료: 00:01:42)
  활성 제작: 1/2 (완료: 00:05:21)
```

### 5.2 알림

- 파견 완료 임박 (1분 전 알림)
- 파견 완료 (강조 알림, 마을 HUD에 알림 점)
- 제작 완료 (소리 + 점)
- 마을 이벤트 발생 (모달)

## 6. gameState

```javascript
// 시간 추적
lastActiveAt: 0,         // 마지막 접속 종료 시각 (epoch ms)
totalPlayTime: 0,        // 누적 접속 시간 (ms, 게임 내 시간)
totalRealTime: 0,        // 누적 실시간 (오프라인 포함)
```

기존 `activeExpeditions`, `activeCrafting`은 자체 startedAt/completesAt 보유.

## 7. 시스템 간 영향

- **파견 시간**: RunSimulator에서 `calcExpeditionTime` 사용 (실시간)
- **제작 시간**: 등급별 고정 (실시간)
- **피로도**: 시간 비례 회복 (실시간)
- **마을 이벤트**: 게임 내 시간 트리거 (접속 중)
- **NPC 호감도/매물**: 거래/대화로 누적, 매물은 게임 내 시간으로 갱신
