# 시스템: 수동 전투 (다키스트 던전 스타일)

> **적용 범위**: BP (Blood Pit) 1차 적용 → 추후 Cargo/BO도 활용 가능.
> **컨셉**: 위치 기반 + 턴제 + 스킬 빌드. 직접 플레이의 깊이 보장.

## 1. 핵심 구조

```
파티 4명 일렬 배치 (포지션 1~4)
적도 일렬 배치 (포지션 1~4)

      [아군 1][아군 2][아군 3][아군 4]  vs  [적 1][적 2][적 3][적 4]
       (전열)              (후열)            (전열)            (후열)
```

- **턴제**: 모든 유닛(아군+적) SPD 비교 → 빠른 순서대로 1회씩 행동
- **라운드**: 모든 유닛 1턴씩 행동하면 1라운드 종료
- **종료**: 한 쪽 전멸 시 전투 종료

## 2. 턴 순서 (SPD 기반)

```javascript
function calcTurnOrder(allUnits) {
  // SPD 높은 순. 같은 SPD는 랜덤
  return allUnits
    .filter(u => u.alive)
    .sort((a, b) => b.spd - a.spd + (Math.random() - 0.5));
}
```

라운드 시작 시 한 번 계산 → 라운드 내에서 그 순서로 진행. (FF 스타일)

대안: 매 턴 재계산 (속도 변화 반영). 일단 라운드 단위.

## 3. 포지션 시스템

### 3.1 배치

- 파티 4명: 포지션 1(가장 앞) → 4(가장 뒤)
- 적 4마리: 포지션 1(가장 앞) → 4(가장 뒤)
- 죽으면 그 자리는 비고, 뒤 유닛이 자동으로 1칸 앞으로 (포지션 시프트)

### 3.2 클래스별 권장 위치

| 클래스 | 권장 포지션 | 이유 |
|---|---|---|
| warrior | 1-2 (전열) | 탱커, 도발 |
| rogue | 1-3 | 근접 dps, 측면 |
| mage | 3-4 (후열) | 원거리 AoE |
| archer | 3-4 (후열) | 원거리 |
| priest | 3-4 (후열) | 힐, 안전 |
| alchemist | 2-3 | 서포터 |

용병이 권장 위치에 있으면 추가 보너스 (예: 도적 1-3 → ATK +10%).

## 4. 스킬 시스템

각 스킬은:
- **사용 가능 위치** (캐스터 포지션): 어느 줄에서 쓸 수 있는지
- **타겟 위치**: 어느 적/아군 포지션을 노릴 수 있는지
- **효과**: 데미지/디버프/힐/위치 변경 등
- **쿨다운**: 라운드 수

### 4.1 클래스별 스킬 (각 2개)

#### 🛡 전사 (warrior)
| 스킬 | 캐스터 위치 | 타겟 | 효과 | 쿨다운 |
|---|---|---|---|---|
| 방패막이 | 1, 2 | 자신 + 인접 아군 | 받는 피해 -40% (1라운드) + 도발 | 3 |
| 강타 | 1, 2 | 적 1, 2 | ATK ×1.5 + 50% 확률로 적 1칸 뒤로 | 2 |

#### 🗡 도적 (rogue)
| 스킬 | 캐스터 위치 | 타겟 | 효과 | 쿨다운 |
|---|---|---|---|---|
| 급소 찌르기 | 1, 2, 3 | 적 1, 2 | 확정 크리 + 출혈 (3라운드 ATK 20%) | 3 |
| 그림자 도약 | 2, 3, 4 | 적 3, 4 | ATK ×1.3 + 자신 1칸 앞으로 | 2 |

#### 🔮 마법사 (mage)
| 스킬 | 캐스터 위치 | 타겟 | 효과 | 쿨다운 |
|---|---|---|---|---|
| 마력 폭발 | 3, 4 | 적 전체 | ATK ×1.5 (AoE) + 화상 | 4 |
| 시간 정지 | 3, 4 | 적 1명 | 다음 1턴 행동 불가 | 4 |

#### 🏹 궁수 (archer)
| 스킬 | 캐스터 위치 | 타겟 | 효과 | 쿨다운 |
|---|---|---|---|---|
| 관통 사격 | 3, 4 | 적 일렬 관통 (1+2 또는 3+4) | ATK ×1.4 (관통) | 3 |
| 저격 | 4 | 적 3, 4 (후열만) | ATK ×2.0 (후열 처리) | 3 |

#### ✝ 사제 (priest)
| 스킬 | 캐스터 위치 | 타겟 | 효과 | 쿨다운 |
|---|---|---|---|---|
| 신성 치유 | 2, 3, 4 | 아군 1명 | HP 30% 회복 | 2 |
| 축복 | 3, 4 | 아군 전체 | ATK +20% (2라운드) | 4 |

#### ⚗ 연금술사 (alchemist)
| 스킬 | 캐스터 위치 | 타겟 | 효과 | 쿨다운 |
|---|---|---|---|---|
| 강화 물약 | 2, 3, 4 | 아군 1명 | ATK +30% (2라운드) | 3 |
| 폭탄 투척 | 2, 3 | 적 2명 (인접) | ATK ×1.4 + 둔화 | 3 |

## 5. 일반 공격

스킬 안 쓰면 일반 공격:
- 캐스터 위치에 따라 타겟 가능한 적 다름
- 전열(1-2): 적 1-2 공격
- 후열(3-4): 적 3-4 또는 원거리 무기 시 1-4 다 가능
- 데미지: `ATK × (1 - def/(def+100))` (현재 percent 모드)

## 6. 위치 변경 메카닉

### 6.1 자동 시프트

유닛 사망 시 뒤 유닛이 1칸 앞으로 자동 이동.

### 6.2 스킬에 의한 위치 변경

- 일부 스킬은 자기/적 위치 변경 (전사 강타: 적 뒤로, 도적 그림자 도약: 자신 앞으로)
- 위치 변경 시 후속 턴 순서/스킬 사용 가능성 변동

### 6.3 후열 보호

전열(1-2)이 모두 살아있으면 후열(3-4)은 일반 공격 받지 않음 (특정 스킬만 통함).

## 7. 상태 효과

| 효과 | 설명 |
|---|---|
| 출혈 | N라운드 동안 매 라운드 ATK × 0.2 피해 |
| 화상 | N라운드 동안 매 라운드 고정 5% HP 피해 |
| 둔화 | SPD -30%, 턴 순서 영향 |
| 도발 | N라운드 동안 적이 도발자를 우선 공격 |
| 보호 | 받는 피해 -40% |
| 행동 불가 | 다음 N턴 행동 못함 (시간 정지 등) |

## 8. UI / 화면 구성

### 메인 화면 (BattleScene 재설계)

```
┌──────────────────────────────────────────────────────┐
│ Round 3   [턴 큐: 워리어 > 적1 > 메이지 > 적2 > ...]  │
├──────────────────────────────────────────────────────┤
│                                                       │
│   [4][3][2][1]  vs  [1][2][3][4]                     │
│   (포지션)            (포지션)                       │
│                                                       │
│   HP바 + 상태 효과 아이콘 (각 유닛 위)               │
│                                                       │
├──────────────────────────────────────────────────────┤
│  현재 행동: 💀 마법사 (포지션 3)                     │
│  [일반 공격] [마력 폭발] [시간 정지] [건너뛰기]      │
│  (스킬 쿨다운 표시)                                  │
└──────────────────────────────────────────────────────┘
```

### 행동 선택 흐름

1. 현재 행동 유닛 강조
2. 스킬 버튼 클릭 → 사용 가능한 타겟 강조 (적/아군)
3. 타겟 클릭 → 행동 실행
4. 애니메이션 (간단한 데미지 팝업, 화면 흔들림)
5. 다음 턴 자동 진행

### 적 턴

- AI가 자동으로 행동 (스킬 또는 일반 공격)
- 우선순위: HP 낮은 적군 우선 / 후열 protect 깰 수 있으면 도전
- 1-1.5초 정도 대기 (관전 시간)

## 9. 보상 / 진행

기존과 동일:
- 클리어 시 골드/XP/장비 드롭
- 라운드 사이 뱀서식 선택지 (zone-bloodpit.md §5)
- 핏 게이지 (zone-bloodpit.md §3)

## 10. 자동 모드 (관전)

### 10.1 컨셉

이건 사실상 **서브 파견의 시각화 버전**. 보상/진행은 ExpeditionManager 기반.

- 출발 게이트에서 "자동 (관전)" 선택
- 서브 파견과 같은 시간 경과형 (백그라운드)
- 다만 BattleScene 진입 시 **시각적으로 진행 중인 전투를 표시**
- 사용자는 단순 관전 (행동 결정 안 함)
- 무한 웨이브가 계속 등장 → 시간이 지나면 결과 (보상/XP) 누적

### 10.2 차이점 (수동 vs 자동)

| | 수동 (다키스트) | 자동 (관전) |
|---|---|---|
| 입력 | 매 턴 스킬/타겟 선택 | 없음 (AI 자동) |
| 시간 | 능동 플레이 5-10분 | 백그라운드 시간 경과형 (1.5분~) |
| 보상 | 메인 보상 (1.5배) | 서브 파견 기본 |
| 진척 | 구역 레벨업 가능 | 깬 레벨까지만 |
| 스킬 | 직접 발동 | AI 자동 발동 (비최적) |

### 10.3 AI 자동 발동 룰

- 쿨다운 회복 즉시 자동 발동
- 우선순위: 사제는 HP 60%↓ 아군 즉시 / 전사는 위기 시 방패막이 / 마법사는 적 다수일 때 폭발

## 13. 작업 마일스톤 (코드)

### Step 1: 코어 (다키스트 엔진)
- DarkestCombat.js — 턴 큐, 스킬 실행, 사망 처리
- 데이터: skills.js 재정의
- 테스트: 콘솔에서 1라운드 시뮬

### Step 2: UI (ManualBattleScene)
- 화면 레이아웃 (4×4 일렬)
- HP바, 상태 효과 표시
- 스킬 버튼, 타겟 선택

### Step 3: BP 통합
- DeployScene "메인 도전" → ManualBattleScene
- 라운드 사이 뱀서식 선택지 통합
- 핏 게이지 통합

### Step 4: 자동 관전 모드
- AutoBattleScene 또는 ManualBattleScene "관전 모드"
- AI 자동 행동
- ExpeditionManager 연동 (백그라운드 진행)

### Step 5: 다른 구역 적용 여부 결정
- Cargo/BO에 동일 시스템 적용할지 결정
- 적용 안 하면 BP만 다키스트, 나머지는 기존

- 보스전 특수 패턴

## 15. 디자인 의도

- **수동 vs 자동의 명확한 차이**: 수동 = 깊이 있는 전략 / 자동 = 방치 보상
- **위치 기반 전략**: 단순 스탯 게임이 아닌 빌드/포메이션 게임
- **다양한 클래스 시너지**: 후열 보호, 위치 변경 등으로 클래스 가치 차별화
- **반복 플레이 동기**: 매 전투마다 다른 스킬 사용 타이밍 = 깊이
