# 시스템: 친화도 트리

> **역할 분리**:
> - **친화도 트리** = 해당 구역/전체에서 적용되는 **패시브 버프** (이 문서)
> - 구역 메커니즘 자체 변경(핏게이지, 열차칸, 저주) = [guild-hall.md](guild-hall.md) F/G/H

## 1. 기본 구조

| 항목 | 값 |
|---|---|
| 대상 | 용병 개인 (구역 트리) / 길드 전체 (공통 트리) |
| 구역 트리 레벨 캡 | 10 |
| 해금 방식 | 포인트제 (레벨당 1pt) |
| 구역 트리 포인트 | 10pt (Lv10) |
| 공통 트리 포인트 | ~7-8pt (길드 Lv8) |
| 노드 수 | 구역당 15개 × 3구역 + 공통 15개 = **60노드** |
| 트리 형태 | 일직선 + 분기 3회 (택1) |
| 캡스톤 | 구역당 2택1, 보스 토큰으로 해금 |
| 노드 효과 | 혼합 (전투 + 경제 + 유틸) |

## 2. 트리 골격 (4트리 공통)

```
[1] → [2] → [3A/3B] → [4] → [5] → [6A/6B] → [7] → [8A/8B] → [9] → [10] → [CA/CB]
 Lv1   Lv2    Lv3      Lv4   Lv5    Lv6      Lv7    Lv8      Lv9   Lv10   Lv10+토큰
```

- 선형 7개 + 분기 6개(3쌍 택1) + 캡스톤 2개(택1) = 15노드
- 구역 트리: 10pt로 11개 선택 가능 → **1개 포기** (선형 하나 or 캡스톤)
- 공통 트리: ~7-8pt로 약 절반만 선택 → 선택 압박 강함
- 분기 선택은 영구. 변경하려면 신전 "재각인 의식" (비용 TBD)

## 3. 친화도 XP 커브 (구역 트리)

```javascript
function affinityXpToNext(currentLevel) {
  const table = [100, 150, 200, 300, 400, 600, 800, 1000, 1500];
  return table[currentLevel - 1];  // Lv1→2 = 100, ...
}
// Lv9 누적: 5,050 XP → Lv10 도달 후 캡스톤 해금 가능
```

파견 1회당 affinityXp ≈ 15-50 (RunSimulator §7).

## 4. 보스 토큰

| 토큰 | 획득 | 용도 |
|---|---|---|
| `bp_token` | Blood Pit 보스 처치 | BP 캡스톤 해금 |
| `cargo_token` | Cargo 보스 처치 | Cargo 캡스톤 해금 |
| `bo_token` | Blackout 보스 처치 | BO 캡스톤 해금 |

보스 토큰은 캡스톤 해금 **전용**. 보스 첫 처치 시 1개 드롭.

```javascript
// 구역 친화도 (용병 개인 — Mercenary 객체 내)
zoneAffinity: {
  bloodpit:  { xp: 0, level: 0, nodes: [], points: 0 },
  cargo:     { xp: 0, level: 0, nodes: [], points: 0 },
  blackout:  { xp: 0, level: 0, nodes: [], points: 0 }
},

// 보스 토큰 (길드 공용)
bossTokens: { bp: 0, cargo: 0, blackout: 0 },

// 공통 트리 (길드 레벨 연동 — gameState 내)
commonAffinityNodes: [],  // 해금한 노드 키 배열

```

## 10. UI 흐름

```
AffinityScene
├─ 탭: BP / Cargo / BO / 공통
├─ 트리 그래프 (해금/대기/잠금 색상)
├─ 노드 호버: 효과 + 잠금 조건
├─ 분기 노드 해금 시 모달: "둘 중 하나를 영구 선택하시오"
└─ 캡스톤: "보스 토큰 1개 + 1pt 소비" 확인
```

## 11. RunSimulator/BattleScene 연동

- 구역 트리: 해당 구역 전투/파견에만 적용
- 공통 트리: 모든 전투/파견에 적용
- `calcPartyPower`에서 양쪽 노드 효과 합산
- 보스 토큰은 `gameState.bossTokens`로 별도 보관
