# 2D 1인개발 전투/파밍 데모 기획서 3종

## 목적
이 문서는 “전투 자체가 재밌는가?”와 “파밍 어트랙션이 반복 플레이를 유도하는가?”를 빠르게 검증하기 위한 2D 소규모 데모 기획 문서이다.

기준:
- 2D 기준
- 1인 개발 가능
- 제작량 최소화
- 반복 플레이 가능
- 파밍 기대감 존재
- 전투 손맛 검증 가능
- 2~4주 내 MVP 가능

공통 개발 조건:
- Unity 2D
- Spine 또는 단순 프레임 애니
- 키보드 + 마우스 조작
- 스테이지 자동 반복 가능 구조
- 최소 UI
- 데이터 중심 설계

---

# 기획안 A — 사냥터 유지형
# 코드명: BLOOD PIT

## 한 줄 설명
좁은 사냥터에서 살아남으며 계속 강해지고, 욕심낼수록 필드가 위험해지는 액션 파밍 게임.

---

# 핵심 재미

## 플레이 감정
- “한 판만 더”
- “조금만 더 먹고 나갈까?”
- “지금 엄청 위험한데 드랍이 좋다”

## 핵심 루프

```text
입장
→ 전투
→ 드랍 획득
→ 강화 선택
→ 필드 위험도 증가
→ 더 강한 적 등장
→ 탈출 or 계속 진행
```

---

# 플레이 구조

## 맵
- 원형 또는 사각형 단일 맵
- 맵 크기 작음
- 오브젝트 최소화

### 맵 요소
- 시체
- 핏자국
- 보급 상자
- 독 웅덩이
- 둥지

맵 자체는 단순하지만 전투 중 계속 상태가 변함.

---

# 전투 시스템

## 플레이어 액션

### 기본 공격

- 히트스톱 존재

### 강공격
- 느리지만 범위 큼
- 넉백 가능
- 다수 처리용

### 회피
- 짧은 무적
- 스태미나 사용

### 상호작용
- 드랍 줍기
- 상자 열기
- 시체 루팅

---

# 적 구조

## 적 종류

### 러너
- 빠름
- 체력 낮음
- 플레이어 압박용

### 브루저
- 느림
- 넉백 저항
- 공간 장악

### 스피터
- 원거리 독탄
- 위치 압박

### 둥지
- 몬스터 생성
- 우선 제거 대상

적은 적지만 조합으로 압박.

# 위험도 시스템

## 핵심 시스템
적을 죽일수록 위험도 증가.

위험도 증가 시:
- 적 체력 증가
- 엘리트 등장
- 독안개 생성
- 드랍률 증가
- 희귀 장비 등장

플레이어는:
- 지금 탈출할지
- 더 욕심낼지
선택.

# 기획안 C — 탈출형 파밍
# 코드명: BLACK OUT

## 한 줄 설명
어두운 시설 내부에서 자원을 파밍하고 살아서 탈출해야 하는 탐욕 중심 액션 게임.

---

# 핵심 감정

- “조금만 더 먹고 나가자”
- “가방이 꽉 찼다”
- “지금 죽으면 다 잃는다”
- “출구까지 갈 수 있을까?”

---

# 핵심 루프

```text
진입
→ 탐색
→ 전투
→ 루팅
→ 가방 관리
→ 위험 증가
→ 탈출
```

---

# 특징

## 중요한 점
전투보다:
- 생존
- 귀환
- 긴장감
이 더 중요.

---

# 맵 구조

## 자동 생성 가능
- 작은 방 연결
- 짧은 플레이 기준
- 반복 플레이 구조

---

# 플레이어 상태

## 가방 무게 시스템
많이 먹을수록:
- 이동속도 감소
- 회피 성능 감소
- 스태미나 감소

욕심낼수록 위험.

---

# 어둠 시스템

## 핵심
빛이 매우 중요.

### 손전등 배터리 존재
배터리 부족 시:
- 시야 감소
- 적 감지 어려움
- 공포감 증가

---

# 전투 시스템

## 기본 철학
싸우는 건 리스크.

가능하면:
- 피하기
- 유인하기
- 빠르게 제거하기
가 중요.

---

# 적 종류

### 스토커

### 차저

### 크리퍼

---

# 파밍 구조

## 아이템 종류

### 고가치 아이템
비싸지만 무거움.

### 경량 아이템
효율 좋음.

### 특수 아이템
탈출 시 영구 해금.

"콘텐츠 양"보다
"전투 순간 감정"이 중요하다.

그래서:
- 피격 반응
- 타격음
- 적 경직
- 드랍 기대감
- 리스크 선택

이 5개가 핵심이다.
